# API Документация

## Базовый URL

Все API эндпоинты находятся по пути `/api/...`

## Формат ответов

### Успешный ответ

```json
{
  "ok": true,
  "data": {...},
  "meta": {
    "duration_ms": 45,
    "count": 10
  }
}
```

### Ошибка

```json
{
  "error": "Описание ошибки"
}
```

## Аутентификация

### POST /api/auth/request-code

Запрос кода для входа на email.

**Тело запроса:**
```json
{
  "email": "user@example.com",
  "name": "Имя пользователя" // опционально
}
```

**Ответ:**
```json
{
  "ok": true,
  "demo_code": "123456" // только если DEV_DELIVERY=true
}
```

**Ошибки:**
- `400` - Неверный email
- `500` - Ошибка БД или отправки email

**Особенности:**
- Если пользователь не существует и указано имя, создается новый пользователь
- Код действителен 10 минут
- В демо-режиме код возвращается в ответе

---

### POST /api/auth/verify-code

Проверка кода и создание сессии.

**Тело запроса:**
```json
{
  "email": "user@example.com",
  "code": "123456"
}
```

**Ответ:**
```json
{
  "ok": true
}
```

**Cookies (устанавливаются автоматически):**
- `sid` - ID сессии (HttpOnly, Secure, SameSite=Lax)
- `u` - JSON с email и name (Secure, SameSite=Lax)

**Ошибки:**
- `400` - Неверный или истекший код
- `500` - Ошибка БД

**Особенности:**
- Сессия действует 7 дней
- Cookie `u` доступен из JavaScript для отображения имени пользователя

---

### POST /api/auth/logout

Выход из системы.

**Ответ:**
```json
{
  "ok": true
}
```

**Особенности:**
- Удаляет сессию из БД
- Очищает cookies `sid` и `u`

## Пользователи (Users)

### GET /api/users

Получить список всех пользователей.

**Ответ:**
```json
{
  "users": [
    {
      "id": 1234567890,
      "name": "Иван Иванов",
      "email": "ivan@example.com",
      "created_at": "2024-01-15T10:30:00.000Z"
    }
  ],
  "meta": {
    "count": 1,
    "duration_ms": 25
  }
}
```

**Особенности:**
- Сортировка по ID (DESC)
- Не требует аутентификации (но защищено middleware)

---

### GET /api/users?id=123

Получить пользователя по ID.

**Параметры:**
- `id` (query) - ID пользователя (целое число)

**Ответ:**
```json
{
  "user": {
    "id": 1234567890,
    "name": "Иван Иванов",
    "email": "ivan@example.com",
    "created_at": "2024-01-15T10:30:00.000Z"
  }
}
```

**Ошибки:**
- `400` - ID не является целым числом
- `null` в поле `user` - пользователь не найден

---

### POST /api/users

Создать нового пользователя.

**Тело запроса:**
```json
{
  "name": "Иван Иванов",
  "email": "ivan@example.com"
}
```

**Ответ:**
```json
{
  "ok": true,
  "id": 1234567890
}
```

**Ошибки:**
- `400` - name и email обязательны
- `409` - Email уже существует

**Особенности:**
- ID генерируется как `Date.now()`
- Email должен быть уникальным

---

### PUT /api/users

Обновить данные пользователя.

**Тело запроса:**
```json
{
  "id": 1234567890,
  "name": "Новое имя", // опционально
  "email": "new@example.com" // опционально
}
```

**Ответ:**
```json
{
  "ok": true
}
```

**Ошибки:**
- `400` - ID не является целым числом или нечего обновлять
- `404` - Пользователь не найден
- `409` - Email уже существует

**Особенности:**
- Можно обновить только name, только email, или оба поля
- Пустые строки игнорируются

---

### DELETE /api/users?id=123

Удалить пользователя.

**Параметры:**
- `id` (query) - ID пользователя

**Альтернативно (тело запроса):**
```json
{
  "id": 1234567890
}
```

**Ответ:**
```json
{
  "ok": true
}
```

**Ошибки:**
- `400` - ID не является целым числом
- `404` - Пользователь не найден

## Заказы (Orders)

### POST /api/orders

Создать новый заказ.

**Требует аутентификации:** Да (cookie `sid`)

**Тело запроса:**
```json
{
  "user": {
    "email": "user@example.com",
    "name": "Имя пользователя"
  },
  "items": [
    {
      "product_id": 1234567890,
      "qty": 2
    },
    {
      "product_id": 1234567891,
      "qty": 1
    }
  ],
  "total_cents": 29998,
  "total_formatted": "299.98 $"
}
```

**Ответ:**
```json
{
  "ok": true,
  "order": {
    "id": 1234567890,
    "user_id": 1234567890,
    "total_cents": 29998,
    "created_at": "2024-01-15T10:30:00.000Z",
    "items_map": {
      "1234567890": {
        "qty": 2,
        "price_cents": 9999,
        "line_total_cents": 19998,
        "name": "Товар 1"
      },
      "1234567891": {
        "qty": 1,
        "price_cents": 9999,
        "line_total_cents": 9999,
        "name": "Товар 2"
      }
    }
  }
}
```

**Ошибки:**
- `401` - Не авторизован
- `400` - items обязательны и не пусты
- `400` - Неверный формат items (product_id и qty должны быть целыми числами)
- `400` - Товары не найдены в БД
- `500` - Ошибка БД

**Особенности:**
- `user` из запроса игнорируется, используется `user_id` из сессии
- Дубликаты `product_id` в `items` суммируются
- Товары проверяются на существование в БД
- `items_map` содержит полную информацию о каждой позиции
- `total_cents` пересчитывается на сервере

## Коды состояния HTTP

- `200` - Успешный запрос
- `201` - Ресурс создан
- `400` - Ошибка валидации
- `401` - Не авторизован
- `404` - Ресурс не найден
- `405` - Метод не разрешен
- `409` - Конфликт (например, дубликат email)
- `500` - Внутренняя ошибка сервера

---

## Логирование

Все запросы логируются в Cloudflare Real-time Logs с метаданными:
- Время выполнения (`duration_ms`)
- HTTP метод и путь
- Статус ответа
- Cloudflare Ray ID
- IP адрес клиента
- User-Agent
