## Общее описание

**edu_rppa** — демонстрационное веб-приложение для мастер-класса по архитектуре информационных систем.

### Технологический стек

- **Frontend**: HTML, CSS, Vanilla JavaScript
- **Backend**: Cloudflare Pages Functions (Serverless Functions)
- **База данных**: Cloudflare D1 (SQLite)
- **Хостинг**: Cloudflare Pages
- **Аутентификация**: Email-код (OTP)

### Основные возможности

- ✅ Управление пользователями (CRUD)
- ✅ Управление товарами (CRUD)
- ✅ Создание заказов с корзиной покупок
- ✅ Аутентификация через одноразовый email-код
- ✅ Защита маршрутов через middleware
- ✅ Сессии на основе cookies

## Архитектура проекта

### Общая схема

```
┌─────────────┐
│   Browser   │ ← Frontend (HTML/CSS/JS)
└──────┬──────┘
       │ HTTP Requests
       ↓
┌──────────────────────┐
│  Cloudflare Pages    │ ← Статические файлы
└──────┬───────────────┘
       │
       ↓
┌──────────────────────┐
│  Cloudflare Functions│ ← Серверная логика (API)
│  (_middleware.js)    │
└──────┬───────────────┘
       │
       ↓
┌──────────────────────┐
│   Cloudflare D1      │ ← База данных (SQLite)
│   (edu_rppa_db)      │
└──────────────────────┘
```

### Поток данных

1. **Пользователь** открывает страницу в браузере
2. **Cloudflare Pages** отдает статические файлы (HTML/CSS/JS)
3. **Frontend** делает запросы к API через `fetch()`
4. **Middleware** проверяет аутентификацию для защищенных маршрутов
5. **Functions** обрабатывают запросы и взаимодействуют с БД
6. **D1 Database** хранит данные (users, products, orders, sessions, auth_codes)

## Основные функции

### 1. Управление пользователями (Users)

- **Просмотр списка** всех пользователей с фильтрацией
- **Создание** нового пользователя (name, email)
- **Просмотр** пользователя по ID
- **Обновление** данных пользователя
- **Удаление** пользователя

**Особенности:**
- Email должен быть уникальным
- Автоматическое создание пользователя при первом входе (если указано имя)

### 2. Управление товарами (Products)

- **Просмотр списка** всех товаров с фильтрацией
- **Создание** товара (name, description, price_cents, image_url)
- **Просмотр** товара по ID
- **Обновление** данных товара
- **Удаление** товара

**Особенности:**
- Цена хранится в центах (целое число)
- Поддержка изображений через URL
- Фильтрация по названию и описанию

### 3. Создание заказов (Orders)

- **Корзина покупок** с выбором количества товаров
- **Автоматический расчет** итоговой суммы
- **Создание заказа** с сохранением в БД
- **Отображение квитанции** после создания заказа

**Особенности:**
- Корзина хранится в памяти браузера (localStorage не используется)
- JSON payload формируется автоматически
- Валидация товаров перед созданием заказа
- Связь заказа с пользователем через сессию

### 4. Аутентификация

- **Запрос кода** на email
- **Верификация кода** и создание сессии
- **Автоматический редирект** на защищенные страницы
- **Выход из системы** с удалением сессии

**Особенности:**
- Одноразовый код действителен 10 минут
- Сессия действует 7 дней
- Два cookie: `sid` (HttpOnly) и `u` (для UI)
- Middleware защищает все маршруты кроме открытых

## API эндпоинты

Подробное описание всех API эндпоинтов находится в файле [API.md](./API.md).

### Краткий обзор

#### Аутентификация
- `POST /api/auth/request-code` - Запрос кода на email
- `POST /api/auth/verify-code` - Проверка кода и вход
- `POST /api/auth/logout` - Выход из системы

#### Пользователи
- `GET /api/users` - Список пользователей
- `GET /api/users?id=123` - Получить пользователя по ID
- `POST /api/users` - Создать пользователя
- `PUT /api/users` - Обновить пользователя
- `DELETE /api/users?id=123` - Удалить пользователя

#### Товары
- `GET /api/products` - Список товаров
- `GET /api/products?id=123` - Получить товар по ID
- `POST /api/products` - Создать товар
- `PUT /api/products` - Обновить товар
- `DELETE /api/products?id=123` - Удалить товар

#### Заказы
- `POST /api/orders` - Создать заказ (требует аутентификации)

## Аутентификация

Подробное описание системы аутентификации находится в файле [AUTHENTICATION.md](./AUTHENTICATION.md).

### Процесс входа

1. Пользователь вводит email (и опционально имя)
2. Система генерирует 6-значный код и отправляет на email
3. Пользователь вводит код
4. Система проверяет код и создает сессию
5. Устанавливаются cookies: `sid` и `u`
6. Пользователь перенаправляется на главную страницу

### Защита маршрутов

- **Открытые маршруты**: `/`, `/login.html`, `/api/auth/*`, статические файлы
- **Защищенные маршруты**: все остальные (проверка через middleware)
